# Protección Jurídica de los Adultos Mayores en Chile

## Leyes y Normas Aplicables al Segmento Objetivo

---

## 1. Ley N° 19.828 — Crea el SENAMA (Servicio Nacional del Adulto Mayor)

| Dato | Detalle |
|------|---------|
| Número | 19.828 |
| Promulgación | 17 de septiembre de 2002 |
| Vigencia | Activa |
| Organismo creado | SENAMA — Servicio Nacional del Adulto Mayor |
| Supervisión | Ministerio de Desarrollo Social y Familia |

### Objetivos del SENAMA (relevantes para el sistema)

El SENAMA fue creado con los siguientes mandatos que **el sistema debe respetar y no contradecir**:

1. Velar por la **plena integración** del adulto mayor en la sociedad
2. **Protegerlo** del abandono y la indigencia
3. **Defender los derechos** que la Constitución y las leyes le reconocen
4. Luchar contra la **discriminación y marginación** de los adultos mayores
5. Fomentar el envejecimiento activo y el desarrollo de servicios sociales

### Implicancias para el Sistema

- El sistema **complementa**, no reemplaza, los servicios del SENAMA
- No puede presentarse como un servicio del SENAMA sin convenio formal
- Debe reforzar la integración y autonomía del adulto mayor, no crear dependencia
- El lenguaje, diseño y contenido deben ser respetuosos y no paternalistas

### Limitación importante de SENAMA
La ley no otorgó a SENAMA facultades específicas de defensa jurídica. Esto significa que **la protección del adulto mayor en contextos digitales depende de normas generales** (Ley del Consumidor, Protección de Datos, etc.) que el sistema debe cumplir con especial rigor.

## 2. Convención Interamericana sobre la Protección de los Derechos de las Personas Mayores (OEA)

| Dato | Detalle |
|------|---------|
| Organismo | Organización de los Estados Americanos (OEA) |
| Ratificación Chile | Sí — publicada en Diario Oficial |
| Carácter | Tratado internacional vinculante para Chile |
| Relevancia | Establece estándar de derechos humanos de personas mayores |

### Derechos Relevantes para el Sistema

La Convención reconoce derechos específicos que el sistema debe respetar:

| Derecho | Descripción | Aplicación al sistema |
|---------|-------------|----------------------|
| **Acceso a la información** | Derecho a recibir información en formatos accesibles | UX accesible; lenguaje simple; opción de texto a voz |
| **Trato digno** | Prohibición de discriminación por edad | Diseño no paternalista; no subestimar las capacidades del usuario |
| **Autonomía y autodeterminación** | Derecho a tomar sus propias decisiones | El sistema orienta, no decide por el usuario |
| **Consentimiento informado** | Especialmente en salud y servicios | Consentimientos claros y comprensibles |
| **No violencia** | Protección contra toda forma de abuso | El sistema no puede usarse para manipular al adulto mayor |
| **Acceso a la justicia** | Garantizar acceso efectivo | El canal de reclamo debe ser simple y accesible |

### Cuidados de Largo Plazo (relevante si el sistema escala)
La Convención define que una persona mayor en situación de dependencia tiene derecho a servicios integrales de calidad. Si el sistema llega a orientar sobre residencias o cuidados, debe asegurarse de informar solo sobre establecimientos regulados y certificados.

## 3. Ley N° 21.013 — Tipifica el Maltrato de Niños, Adultos Mayores y Personas con Discapacidad

| Dato | Detalle |
|------|---------|
| Número | 21.013 |
| Materia | Tipifica maltrato de personas mayores |
| Relevancia | **ALTA** — El sistema no puede ser usado como vector de maltrato o engaño |

### Implicancias para el Sistema

- El sistema NO puede diseñarse para manipular decisiones económicas del adulto mayor
- No puede inducir al usuario a contratar servicios innecesarios o a entregar dinero
- El personal administrativo que acceda a datos de usuarios adultos mayores debe ser capacitado sobre esta ley
- Si el sistema detecta señales de maltrato en las consultas del usuario, debe orientar a recursos de ayuda (SENAMA, PDI, Carabineros)

## 4. Principios Rectores para el Diseño del Sistema con Adultos Mayores

Basados en la Convención OEA y las leyes chilenas, el sistema debe cumplir estos principios:

### Principio de Dignidad
El sistema nunca debe minimizar, ridiculizar ni subestimar al usuario adulto mayor. El lenguaje del chat IA debe ser respetuoso, paciente y no condescendiente.

### Principio de Autonomía
El sistema entrega información y opciones; el usuario decide. La IA no puede decir "usted debe hacer X" sino "sus opciones son A, B o C".

### Principio de No Discriminación
El sistema debe ser igualmente útil independientemente del nivel educacional, origen étnico o situación económica del usuario.

### Principio de Accesibilidad Universal
- Tipografía mínima 18px
- Contraste alto (WCAG AA mínimo, idealmente AAA)
- Lenguaje simple (nivel educacional básico como referencia)
- Posibilidad de texto a voz
- Sin tecnicismos sin explicación

### Principio de Consentimiento Informado Reforzado
Para adultos mayores, los consentimientos deben ser especialmente claros:
- Lenguaje simple, sin términos legales oscuros
- Tamaño de letra legible
- Opción de imprimir o guardar el documento de consentimiento
- No usar tiempos de respuesta cortos que generen presión

## 5. Consideraciones Especiales para Datos de Adultos Mayores

### Capacidad Legal
- La mayoría de los adultos mayores tienen plena capacidad jurídica
- El sistema **no debe asumir incapacidad** por la edad
- Si el sistema detecta señales de deterioro cognitivo severo, no debe presionar al usuario; debe sugerir consultar con un familiar o profesional

### Privacidad Reforzada
Los datos de adultos mayores deben tratarse con especial cuidado porque este grupo es frecuentemente blanco de fraudes y estafas. El sistema debe:
- No pedir más datos de los necesarios
- Explicar claramente por qué se pide cada dato
- Dar opciones para no proporcionar datos opcionales
- Nunca compartir datos con terceros sin consentimiento explícito

### Canales de Apoyo
El sistema debe siempre tener acceso visible a:
- SENAMA: 800 400 035 (fono mayor, gratuito)
- Chile Atiende: 101
- Carabineros: 133
- PDI (en caso de fraude o maltrato): 134
